Biomass Ontology: Description back to ToC

The Biomass ontology represents the hierarchy of the biomass. Figure 1 represets an overview of the main classes, data properties, object properties and instances of the biomass ontology For the clarity of the figure, in some cases we reuse abrevation. For example to refer to the "Blends and Mixtures" type that belongs to the " Orchard And Horticulture Fruit" group we call it "BlendsAndMixturesOAHF". We take the first letter of each word in the group in upper case

Instantiating Biomass ontology back to ToC

To represent the biomass hierarchy, we use the skos relations (skos:narrower and skos:broader).
To specify that one concept has a more generic meaning than another, we use the skos:broader relation. For example, to represent that biom:WoodyBiomass is more generic than biom:PlantationandForest we define the skos:broader relation as : biom:PlantationandForest skos:broader biom:WoodyBiomass.
To specify that one concept has a more specific meaning than another, skos:narrower is used.
